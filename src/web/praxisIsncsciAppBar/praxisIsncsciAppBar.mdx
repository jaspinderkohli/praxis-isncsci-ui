# Praxis ISNCSCI App Bar

## Design Problem

The application needs an app bar that allows the use of the following optional subcomponents:

- Menu button (hamburger icon button)
- Title
- Actions (buttons like "calculate", "download", etc.)

## Approach

We are favoring composition so we add slots for the optional subcomponents.
This will make it easy for the component or controller adding them to control their appearance and subscribe to their events.

### Slots

- `menu-button` - Expects a button with a hamburger icon
- `title` - Expects an element with text
- `actions` - Expects a container, preferably a `<div>` with buttons
